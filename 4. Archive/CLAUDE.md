# 4. Archive — 운영 가이드

## 폴더 목적

**완료되거나 더 이상 활성화되지 않는 모든 항목**을 보관합니다.  
삭제하지 않고 보존합니다. 검색 가능하고, 재활성화할 수 있습니다.

## 어떤 항목이 들어오는가

- 완료된 프로젝트 폴더 (`1. Projects/` → 여기)
- 비활성화된 영역 폴더 (`2. Areas/` → 여기)
- 더 이상 참고하지 않는 리소스 (`3. Resources/` → 여기)
- 폐기된 초안, 실패한 실험 노트

**들어오지 않는 것**
- 영구 지식 노트 — `5. Zettelkasten/20. Permanent/`의 노트는 Archive 대상이 아님 (지식은 영구 보존)

## 파일 구조

```
4. Archive/
├── CLAUDE.md
├── Projects/                   # 완료된 프로젝트
├── Areas/                      # 비활성화된 영역
└── Resources/                  # 폐기된 리소스
```

## Archive 이동 절차

1. 해당 노트/폴더의 `status`를 `completed` 또는 `archived`로 변경
2. `updated` 날짜를 현재 날짜로 갱신
3. 폴더/파일을 `4. Archive/[원래 카테고리]/`로 이동
4. 해당 허브 노트에서 링크가 깨지지 않는지 확인 (Obsidian이 자동 업데이트)

## 재활성화

Archive에서 다시 꺼내올 때는 원래 위치(`1. Projects/`, `2. Areas/` 등)로 이동하고 `status`를 적절히 변경합니다.

## 작업 규칙

1. **삭제 지양**: 완료된 프로젝트도 Archive에 보존 — 과거 컨텍스트는 나중에 가치 있음
2. **원본 구조 유지**: 이동 시 내부 폴더 구조를 그대로 유지
3. **정기 정리 금지**: Archive는 적극적으로 정리하지 않음, 검색으로 찾기
4. **링크 유지**: Obsidian의 자동 링크 업데이트를 활용 (파일 이동 시 자동 처리)
