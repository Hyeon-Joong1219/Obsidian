# 5. Zettelkasten/00. Inbox — 운영 가이드

## 폴더 목적

**빠른 아이디어 캡처와 임시 메모**를 위한 수집함입니다.  
생각이 떠오른 즉시 기록하는 장소입니다. 완성도를 신경 쓰지 않습니다.  
**반드시 주 1회 이상 정리**하여 Inbox가 쌓이지 않도록 합니다.

## 어떤 노트가 들어오는가

- 즉흥적 아이디어, 메모
- 나중에 처리할 링크, 인용문
- 빠른 일지, 단상
- 아직 어디 분류할지 모르는 모든 것

**Inbox는 최종 목적지가 아닙니다** — 모든 노트는 처리 후 적절한 위치로 이동합니다.

## 처리 흐름 (주 1회 Inbox 정리)

```
Inbox 노트 검토
    ↓
이 생각이 영구적 지식인가?
    ├─ YES → 5. Zettelkasten/20. Permanent/로 이동 (퍼머넌트 노트 작성)
    ├─ 책/글에서 온 내용인가?
    │      └─ YES → 5. Zettelkasten/10. Literature/로 이동
    ├─ 실행할 작업인가?
    │      └─ YES → 1. Projects/ 또는 2. Areas/로 이동
    ├─ 참고 자료인가?
    │      └─ YES → 3. Resources/로 이동
    └─ 필요 없는가? → 삭제
```

## Inbox 노트 형식

형식에 구애받지 않습니다. 최소한 아래만 포함:

```yaml
---
created: YYYY-MM-DD
---
```

내용은 단 한 줄이어도 됩니다.

## 작업 규칙

1. **즉시 기록**: 완성도보다 속도. 나중에 다듬음
2. **주 1회 정리**: 매주 Inbox를 비우는 루틴 확립 (Inbox Zero)
3. **분류 기준 없음**: Inbox에 넣을 때 분류를 고민하지 않음
4. **장기 체류 금지**: 2주 이상 Inbox에 머무는 노트는 강제로 이동하거나 삭제
5. **일일 노트와 구분**: 일일 노트(Daily Note)는 별도 관리, Inbox는 주제별 메모용
